This firmware runs on a second, separate microcontroller physically located at the ground station and serves a single, narrowly-scoped purpose: it is the only node in the system with a direct hardware view of the radio link itself, and its job is to receive each incoming LoRa packet, annotate it with link-quality metrics that only the receiving radio can measure, and forward the result verbatim over a USB serial connection to the ground computer. Because signal strength and signal-to-noise ratio are properties of reception rather than transmission, this information physically cannot originate anywhere except at this node, and the firmware appends both values to the tail of the CSV telemetry string immediately after reading them from the radio hardware, before the packet is passed downstream. Deliberately, this firmware contains no telemetry parsing, no display logic, and no interpretation of the payload it is relaying; it is intentionally a thin, low-risk bridge, which minimizes the amount of code standing between the rocket's radio and the ground computer's serial port and correspondingly minimizes the number of places a bug in this layer could silently corrupt or delay the data.
